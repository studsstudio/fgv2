## Running locally

### Frontend only (no server)
Open `index.html` directly in any browser. Self-contained, works via `file://`.

### Full stack (with auth/payments/snapshots)
```bash
npm install
npm start
# Server runs on http://localhost:5000
```

Set env vars: `DATABASE_URL`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `SESSION_SECRET`, `ADMIN_USER_ID`.

## Build

After editing any `.js`/`.css`/`shell.html`:
```bash
python build.py
```
This regenerates `index.html` with everything inlined.

⚠️ Running `build.py` overwrites `index.html`.

## Conductor architecture

The conductor is the single authority over how arcs, live sets, DJ brain, touch gestures, form cycle, and ambient drift influence the mix. Every system submits *intents* to the resolver, which picks the highest-priority active claim per target.

### Priority ladder

| Source         | Priority |
|----------------|----------|
| userGesture    | 120      |
| arc            | 100      |
| liveSet        | 80       |
| djBrain        | 50       |
| formCycle      | 30       |
| microTexture   | 20       |
| ambient        | 10       |

### Submitting intents

```js
submitConductorIntent('source.tag', 'reverbAmt', 1.8, CONDUCTOR_PRIORITY.liveSet, 30000);
clearConductorIntents('arc.');  // release all 'arc.*' claims
```

### Locks

```js
conductorLocks.arc      // arc active → blocks structural decisions
conductorLocks.genre    // transition in progress → blocks macro drift
conductorLocks.drums    // drum pattern mutation locked
conductorLocks.density  // mute/unmute toggles locked
```

### Taste rules (DJ Brain)

- During arc: structural decisions vetoed
- No ADD/REMOVE within 16 bars of previous structural change
- After drop: 16-bar cooldown
- After break: 8-bar cooldown
- Genre drift deferred (up to 8 retries, ~32 bars) until arc is idle

## Feature flags (entitlements)

The app reads the user's plan from the server on boot:
1. `fg-auth.js` calls `GET /api/auth/user` and `GET /api/subscription`
2. If `isAdmin` or subscription `isActive` → `FG.plan = 'pro'`
3. Else if logged in → `FG.plan = 'member'`
4. Else → `FG.plan = 'free'`

For local testing without a server, you can still override via localStorage:
```js
localStorage.setItem('fg_plan', 'pro');     // then reload
```

In code:
```js
if (!FG.can('saveField')) { FG.prompt('saveField'); return; }
```

## Auth / membership UI

`fg-auth.js` (ported from v1) handles:
- **Header right chip** — Sign In / Sign Up buttons OR user avatar + plan badge
- **Auth modal** — single modal toggles between sign-in and sign-up
- **Upgrade modal** — Stripe checkout entry with feature list and pricing
- **Manage Subscription** — opens Stripe customer portal
- **Admin panel** — full stats dashboard (users, sessions, country breakdown)
- **Session tracking** — `POST /api/session/start` on auth, `sendBeacon` to `/api/session/end` on tab close
- **Checkout return handling** — green success banner on `?checkout=success`

State lives on `window.appState`:
```js
appState.user       // null when signed out, user object when signed in
appState.isPro      // true if subscription.isActive
appState.isAdmin    // true if server marks the user as admin
appState.sessionId  // current analytics session id
```

## Debug telemetry

Append `?debug=1` to the URL.

```js
fgDump()                 // recent DJBrain decisions, journey/chapter, arcs
fgClaims()               // active conductor intents per key
fgElements()             // every element + label state (label-debug tool)
fgCompositionTemplate    // current Suprematist composition name
```

Watch for `djDecisionVeto`, `genreDriftDeferred`, `genreDriftQueued`, `genreDriftDrained` events.

## iOS Safari notes

- All `localStorage` access is wrapped in `try/catch` (Private Browsing throws SecurityError on every access)
- Global error handler shows banners on screen (red = real error, orange = opaque/cross-origin)
- Tap any error banner to dismiss
